Getting started

Before the extension can detect anything, it needs to know your address. Open the Addresses tab and add your current address. That's the only setup required.

Once set, the extension scans every page you visit in the background. When it finds your address, that site is added to the Sites tab automatically — no manual input needed.

First time checklist

  1. Add your current address in the Addresses tab.
  2. Browse normally — visit your bank, utilities, insurance, subscriptions.
  3. Check the Sites tab after a few days to see what's been detected.
  4. When you move, click Start move on the Dashboard.

How detection works

On each page you visit, the extension scans visible text and pre-filled form fields. It normalises the text and looks for your address using common abbreviations — so "Street" matches "St", "South Australia" matches "SA", and so on.

Detection happens entirely on your device. The extension never sends page content anywhere.

When a site is not detected

Some sites display addresses in unusual formats — truncated, reordered, or with extra text around them. If a site has your address but it's not being detected, the fix is to add a variant.

You can also right-click any address text on a page and use the Address Tracker context menu to add the site manually.

Detection settings

You can tune detection behaviour in the Settings tab. See the Settings section for what each toggle does.

Address variants

A variant is an alternate form of your address — exactly as it appears on a specific site. Common abbreviations are matched automatically. Variants are for anything else.

Example: Your address is 12 Smith Street, Perth WA 6000. A site shows it as Smith St 12, Perth, 6000 WA. That won't match automatically — add it as a variant and it will.

How to add a variant

From the page (recommended): Select the address text on the site, right-click, and choose Add "…" as new variant…. The management page opens with the text pre-filled. Review it and click Add.

From the Addresses tab: Open the Addresses tab, find your address, and click + Add variant.

Matching existing variants

If you select text on a page and it matches a variant you've already added, right-click and pick that variant from the list. This records the page without adding a duplicate.

When you move

Click Start move on the Dashboard and enter your new address. The extension uses the Sites list as your update checklist.

Working through the checklist

The Dashboard shows two columns: Needs Update and Done. Visit each site, update your address, then mark it done — either from the Dashboard or from the on-page banner.

The progress bar tracks how many sites are done out of the total. The toolbar badge shows the remaining count at a glance.

The on-page banner

When you visit a site that still has your old address, a small banner appears in the top-right corner. It shows your new address so you can copy it, and lets you mark the site as done or dismiss the banner for that session.

The banner can be turned off in Settings if you don't want it.

Completing the move

Once all sites are marked done, click Complete move on the Dashboard. Your new address becomes current, the old one is archived, and the checklist clears.

If you need to stop mid-move, click Cancel move. The new address is discarded and your old address stays current.

Off-web tasks

Some address updates don't have a website — calling HR, visiting a post office, updating a driver's licence. Add these as tasks from the Sites tab using + Add task. They appear in the checklist alongside your detected sites.

Actions

Note
Add a private note to a site — useful for remembering login details or special instructions.
Ignore
Hide this site from the list. It stops being monitored. You can restore ignored sites from the Settings tab.
Delete
Permanently remove the site and all its history from the extension. Cannot be undone.

Settings

Detection

Scan visible page text
The main detection method. Walks the rendered page and looks for your address in visible text. Turn this off only if you want to rely entirely on form field scanning.
Scan pre-filled form values
Checks input and textarea fields for your address. Useful for account settings pages where your address is already filled in but not displayed as text.
Skip footers and headers
Ignores text in page headers and footers. Reduces false positives from sites that print your address in a footer on every page.
Re-scan when the page changes
Watches for DOM changes and re-scans when content loads dynamically. Enable this for single-page apps (SPAs) where navigation doesn't reload the page.
Show on-page banner during a move
Shows the update banner when your old address is found on a page during a move. Disable if you find it disruptive.

Backup and transfer

Export saves your addresses, sites list, and move history as a JSON file. Use this to back up your data or move it to another device.

Import loads a previously exported file. You can either merge it with your existing data (adds new entries, keeps existing ones) or replace everything.

The export file contains your home address and a list of every site the extension has found it on. Keep it private.

Ignored sites

Sites you've ignored appear here. Click Restore to bring one back into the active list.

Privacy

Everything the extension stores stays on this device. No data is sent to any server. The extension makes no network requests of its own.

The extension reads page content only to search for your address. It does not read passwords, payment details, or anything unrelated to address detection.

If you uninstall the extension, all stored data is deleted by the browser automatically. Export first if you want to keep your sites list.
